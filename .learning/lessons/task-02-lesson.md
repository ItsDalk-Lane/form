## 核心概念讲解

### 1. 什么是 Node.js？

**Node.js 是 JavaScript 的运行环境。**

**通俗解释**：
- JavaScript 最初只能在浏览器里运行
- Node.js 让 JavaScript 可以在电脑上直接运行（不需要浏览器）
- 它是现代前端开发的必备工具

**为什么需要它？**

| 用途 | 说明 |
|------|------|
| 运行构建工具 | 把 TypeScript 编译成 JavaScript |
| 管理依赖包 | 使用 npm 安装项目需要的库 |
| 运行脚本 | 执行 `npm run build` 等命令 |
| 开发服务器 | 本地预览和调试 |

**打个比方**：
- 浏览器是 JavaScript 的"教室"（只能在里面运行）
- Node.js 给 JavaScript 造了一个"操场"（可以在电脑上自由运行）

### 2. 如何检查 Node.js 是否安装？

打开终端（命令行），输入：

```bash
node --version
```

如果显示版本号（如 `v20.10.0`），说明已安装。

输入：

```bash
npm --version
```

检查 npm 是否安装（npm 随 Node.js 一起安装）。

### 3. VS Code 界面介绍

VS Code 是我们主要的代码编辑器，界面分为几个区域：

```
┌─────────────────────────────────────────┐
│  文件  编辑  查看...      □ □ ✕        │  ← 顶部菜单栏
├───────┬─────────────────────────────────┤
│       │                                 │
│  📁   │      编辑器区域                  │
│ 文件   │    （代码显示在这里）            │
│ 浏览器  │                                 │
│       │                                 │
│       ├─────────────────────────────────┤
│       │  终端面板（可选）                │
├───────┴─────────────────────────────────┤
│  状态栏（行号、文件类型、Git状态等）       │  ← 底部状态栏
└─────────────────────────────────────────┘
```

**主要区域说明**：

| 区域 | 位置 | 作用 |
|------|------|------|
| 活动栏 | 最左侧 | 切换视图（文件、搜索、Git等） |
| 文件浏览器 | 左侧 | 显示项目文件夹和文件 |
| 编辑器 | 中间 | 编辑代码的主要区域 |
| 终端 | 底部 | 运行命令行指令 |
| 状态栏 | 底部 | 显示当前文件信息和状态 |

### 4. 如何在 VS Code 中打开项目？

**方法一：通过菜单**
1. 打开 VS Code
2. 点击 "文件" → "打开文件夹"
3. 选择项目文件夹（`c:\Desktop\插件开发\.obsidian\plugins\form\`）
4. 点击"选择文件夹"

**方法二：通过终端**
1. 打开终端
2. 进入项目目录：`cd c:\Desktop\插件开发\.obsidian\plugins\form\`
3. 输入：`code .`

**方法三：右键菜单（推荐）**
1. 在文件资源管理器中找到项目文件夹
2. 右键点击 → "通过 Code 打开"

## 实践任务

### 任务 2.1：检查 Node.js 安装

1. **打开终端**
   - Windows：按 `Win + R`，输入 `cmd`，回车
   - 或在 VS Code 中：点击菜单 "终端" → "新建终端"

2. **输入命令检查版本**
   ```bash
   node --version
   ```
   如果显示版本号（如 `v18.x.x` 或 `v20.x.x`），说明已安装。

3. **检查 npm 版本**
   ```bash
   npm --version
   ```

**如果显示"不是内部或外部命令"，说明需要安装 Node.js。**

### 任务 2.2：安装 Node.js（如未安装）

1. 访问官网：https://nodejs.org/
2. 下载 **LTS（长期支持）版本**
3. 运行安装程序，一路点击"下一步"
4. 安装完成后，重新打开终端，再次检查版本

### 任务 2.3：在 VS Code 中打开项目

选择你喜欢的方法，在 VS Code 中打开项目文件夹。

打开后，你应该能在左侧看到文件树：
- .learning/
- node_modules/
- src/
- manifest.json
- package.json
- ...

### 任务 2.4：熟悉 VS Code 界面

1. **点击左侧图标**
   - 第一个图标（文件）：显示文件树
   - 第二个图标（搜索）：搜索文件内容
   - 第三个图标（Git）：查看代码变更

2. **打开一个文件**
   - 在文件树中点击 `manifest.json`
   - 观察编辑器区域显示的内容

3. **打开终端面板**
   - 点击菜单 "终端" → "新建终端"
   - 或按快捷键 `` Ctrl + ` ``（数字1左边的键）
   - 在终端中输入 `pwd`（查看当前路径）

### 任务 2.5：运行第一个命令

在 VS Code 终端中输入：

```bash
npm run build
```

观察会发生什么：
- 终端会显示构建过程
- 项目根目录会生成/更新 `main.js` 文件
- 这就是 TypeScript 编译成 JavaScript 的过程

## 常见问题

### Q: 为什么需要安装 Node.js？
A: 因为我们需要它来：
- 编译 TypeScript 代码
- 管理项目依赖
- 运行构建脚本

### Q: npm 是什么？
A: npm（Node Package Manager）是 Node.js 的包管理工具，用来：
- 安装第三方库（如 `obsidian` API）
- 运行脚本命令（如 `npm run build`）

### Q: 安装 Node.js 后出现警告？
A: 某些警告是正常的，只要 `node --version` 能显示版本号即可。

### Q: VS Code 中找不到终端？
A: 点击顶部菜单 "终端" → "新建终端"，或使用快捷键 `` Ctrl + ` ``
