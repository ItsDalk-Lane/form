# 📖 Obsidian 插件学习辅导系统使用说明

欢迎使用这个为零基础学习者设计的学习辅导系统！

---

## 🎯 系统目标

通过循序渐进的 52 个任务，帮助你从零开始掌握：
- HTML 与 CSS（网页结构和样式）
- JavaScript（编程基础和交互逻辑）
- TypeScript（类型安全的 JavaScript）
- React（现代前端框架）
- Obsidian 插件开发（将知识应用到实践）

---

## 📂 系统文件说明

### 核心文件

1. **progress.json**
   - 作用：追踪你的学习进度
   - 内容：当前任务、已完成任务、任务历史记录

2. **knowledge-base.json**
   - 作用：记录你已掌握的知识点
   - 内容：概念分类、掌握状态、首次学习时间

3. **current-task.md**
   - 作用：显示当前学习状态和下一步行动
   - 内容：当前任务信息、学习进度概览

### 存档文件夹

4. **lessons/**
   - 作用：保存每次课程的教学内容
   - 命名：`task-XX-lesson.md`（XX为任务编号）

5. **code-reviews/**
   - 作用：保存每次代码审阅的记录
   - 命名：`task-XX.md`（XX为任务编号）

---

## 🔄 学习流程

### 1. 开始新任务
告诉我："**我要开始任务X**"（X为任务编号）

### 2. 接受教学
我会为你提供：
- 📚 学习目标
- 🧠 核心概念讲解
- 💡 概念示例（仅用于理解，不可复制到项目）
- 🎯 实践任务说明
- ✅ 完成标准

### 3. 动手实践
根据教学内容，自己编写代码完成任务

### 4. 提交代码
完成后告诉我：
- 你的代码在哪个文件（提供路径）
- 你遇到了哪些困难
- 你对结果的理解

### 5. 接受审阅
我会检查你的代码，提供：
- ✅ 做得好的地方
- 📋 规范检查结果
- 🔧 需要改进的地方
- 💪 改进建议（引导你思考，不直接给代码）

### 6. 改进代码
根据反馈修改代码

### 7. 确认通过
通过后，我会：
- 更新学习进度
- 记录新掌握的知识点
- 允许你开始下一个任务

---

## 📋 规范要求

在整个学习过程中，我会严格遵循两份规范文档：

1. **Obsidian 插件开发原则**
   - 路径：`.learning/Obsidian 插件开发原则.txt`
   - 内容：插件架构、模块化、生命周期等规则

2. **通用代码规范**
   - 路径：`.learning/代码规范原则.txt`
   - 内容：命名、格式、注释、SOLID 原则等

你的代码也需要遵循这些规范，我会在审阅时检查。

---

## 🎓 教学原则

### 我会做的事：
✅ 详细讲解概念和原理
✅ 提供概念示例帮助理解
✅ 引导你思考解决方案
✅ 审阅代码并提供反馈
✅ 追踪你的学习进度和知识掌握情况

### 我不会做的事：
❌ 直接给你可复制到项目的代码
❌ 替你写代码
❌ 重复讲解你已掌握的知识（除非需要复习）
❌ 跳过必要的学习步骤

## 💡 学习建议

1. **按顺序完成任务**
   - 每个任务都有前置要求
   - 跳过任务会导致知识断层

2. **遇到困难是正常的**
   - 编程学习就是不断试错的过程
   - 告诉我你的困难，我会帮你分析

3. **多动手实践**
   - 光看不练是学不会的
   - 尝试修改代码，看看会发生什么

4. **保持代码整洁**
   - 从第一天起就养成良好的编码习惯
   - 遵循规范比完成功能更重要

5. **做笔记**
   - 记录你的理解和心得
   - 复习时会很有帮助
