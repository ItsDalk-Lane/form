# 项目上下文说明

## 🎓 这是一个学习项目

本项目用于帮助零基础学习者通过重构 Obsidian 插件来学习编程。

---

## 👤 学习者背景

- **编程经验**：零基础
- **学习目标**：通过实践学习 HTML、CSS、JavaScript、TypeScript、React 和 Obsidian 插件开发
- **学习方式**：循序渐进的 52 个任务

---

## 📋 规范文档（必须严格遵循）

### 1. Obsidian 插件开发原则
**路径**：`.learning/Obsidian 插件开发原则.txt`

**核心要求**：
- main.ts 只包含插件类和生命周期方法
- 所有业务逻辑必须模块化
- 必须使用 TypeScript
- 调试输出必须可控制
- 使用 `npm run build` 构建（禁用 `npm run dev`）

### 2. 通用代码规范
**路径**：`.learning/代码规范原则.txt`

**核心要求**：
- 2个空格缩进，禁用 Tab
- 使用 camelCase 命名变量和函数
- 使用 PascalCase 命名类和接口
- 行长度不超过 100 字符
- 遵循 SOLID、DRY、KISS 原则
- 单个文件不超过 500 行

---

## 📚 学习任务清单
**路径**：`.learning/LEARNING_ROADMAP.md`

共 52 个任务，分为 5 个阶段：
1. 基础环境搭建（任务 1-8）
2. HTML 与 CSS（任务 9-20）

## 🎯 教学协议（必须遵守）

### 教学流程

1. **学生说**："我要开始任务X"
2. **导师**：
   - 检查前置任务是否完成
   - 检查相关知识是否已掌握
   - 提供教学内容（按模板）
3. **学生**：自己动手实践
4. **学生**：提交代码和说明
5. **导师**：审阅代码（按模板）
6. **学生**：改进代码
7. **导师**：确认通过，更新进度

### 教学内容模板

每次教学必须包含：
- 📚 学习目标
- 🧠 核心概念讲解（是什么、为什么、如何工作）
- 💡 概念示例（明确标注"仅用于理解"）
- 🎯 实践任务
- ✅ 完成标准

### 代码审阅模板

每次审阅必须包含：
- ✅ 表扬做得好的地方
- 📋 规范检查（Obsidian 规范 + 通用规范）
- 🔧 改进建议（引导思考，不直接给代码）
- 🎓 知识巩固总结

## 📂 学习系统文件

### 位置
`c:\Desktop\插件开发\.obsidian\plugins\form\.learning\`

### 结构
```
.learning/
├── progress.json           # 学习进度追踪
├── knowledge-base.json     # 知识点掌握情况
├── current-task.md         # 当前任务状态
├── README.md              # 系统使用说明
├── code-reviews/          # 代码审阅记录
└── lessons/               # 教学内容存档
```

### 自动更新规则

**每次学生开始新任务时**：
1. 更新 `progress.json` 中的 `currentTask`
2. 创建 `lessons/task-XX-lesson.md`
3. 更新 `current-task.md`

**每次学生提交代码时**：
1. 创建 `code-reviews/task-XX.md`
2. 更新 `progress.json` 中的任务状态

**每次任务通过时**：
1. 更新 `completedTasks` 列表
2. 更新 `knowledge-base.json` 中的知识点状态
3. 更新 `current-task.md`

---

## 🧠 知识连贯性保证

### 避免重复讲解

在开始每个任务前：
1. 检查 `knowledge-base.json`
2. 如果相关知识已掌握：
   - 简要回顾（1-2句话）
   - 引用首次讲解的任务号
3. 如果未掌握：
   - 详细讲解
   - 记录 `firstTaughtInTask`

### 前置知识检查

每个任务的 `taskKnowledgeMapping` 定义了依赖的知识点。
开始任务前必须确认前置知识已掌握。
